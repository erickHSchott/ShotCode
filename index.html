Guía Maestra: Capa de Datos HTML5

Estudia estos conceptos y experimenta con el código en vivo. En el Quiz Final, esta guía será bloqueada.

1. IDENTIDAD Y SELECTORES:
  • El id se representa con # y es único. Es el "nombre" que JavaScript usa para extraer datos.
  • La class se representa con . y es grupal. Se usa para estilos CSS compartidos.
  • El ID siempre tiene prioridad sobre la Clase.
2. INPUT TYPES (TECLADOS TÁCTILES):
  • number: Teclado numérico.
  • tel: Teclado de dígitos grandes (ideal para códigos).
  • range: Control deslizante (slider).
  • date: Calendario nativo; color: Selector de color.
🔬 Lab 1: Prueba los tipos (Cambia 'number' por 'range', 'color' o 'date')
3. ATRIBUTOS Y VALIDACIÓN:
  • min/max: Límites físicos.
  • step: Incrementos (ej. 0.5 permite decimales).
  • readonly: Se ve pero no se edita (sí se envía).
  • disabled: Se bloquea totalmente (no se envía).
🔬 Lab 2: Limites y Saltos (Cambia max="100" o borra 'readonly')
4. ESTRUCTURA Y DOM:
  • div: Contenedor genérico; section: Contenedor temático.
  • Padre e Hijo: Si ocultas al Padre, el Hijo desaparece.
  • Contenedor Dinámico: Elemento vacío con un ID donde JS inyecta datos.

Referencia Rápida

  • #ID: Único. Gana en prioridad. JavaScript.
  • .Class: Grupal. Estilos CSS.
  • type="tel": Teclado numérico de dígitos grandes.
  • type="range": Control deslizante / Slider.
  • readonly vs disabled: Readonly sí se envía, disabled no.
  • Padre/Hijo: Borrar al padre elimina al hijo.
