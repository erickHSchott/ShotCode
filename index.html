📚 Guía Maestra: Fundamentos de Desarrollo Web

1. La Estructura Atómica (Tags y Atributos)

Anatomía de una Etiqueta: Casi todas las etiquetas tienen apertura <etiqueta> y cierre </etiqueta>. Los Atributos son propiedades que se escriben dentro de la apertura para dar instrucciones extra. <a href="link">Texto</a>
Comentarios y Notas: Para dejar notas que el navegador no lea, usamos <!-- esto es un comentario -->. Es vital para organizar tu código.

2. El Sistema de Identidad (ID vs Class)

ID (ADN Único): Solo puede existir un ID con el mismo nombre en toda la página. Se usa para elementos únicos como el título principal. En CSS se llaman con #.
Class (Uniforme): Puedes repetir la misma clase en muchos elementos. En CSS se llaman con un punto .. El ID siempre es "más fuerte" que la clase (especificidad).

3. UX y Formularios (Inputs y Validación)

Tipos de Input: El type define qué aparece: checkbox (casillas), date (calendario), tel (teléfono), email o submit (botón de envío). <input type="checkbox">
Validadores Automáticos: required obliga a llenar el campo. placeholder pone un texto sugerido. min pone un límite numérico y maxlength limita el texto.

4. Arquitectura Semántica (El orden de Google)

Etiquetas de Organización: <main> es el corazón de la web. <nav> para menús. <section> y <article> para agrupar temas. <aside> para barras laterales y <footer> para el cierre.
El "Cerebro" (Head): En el <head> vive el <title> de la pestaña, la codificación UTF-8 (para tildes y Ñ) y el Viewport para que la web sea responsiva (se vea bien en móviles).

5. Multimedia y Datos

Enlaces e Imágenes: Los enlaces <a> usan href. Si agregas target="_blank", abres en pestaña nueva. Las imágenes <img> deben llevar alt (texto alternativo) para ser accesibles.
Tablas: Se crean por filas <tr> (Table Row) y dentro llevan celdas <td> (Table Data). Solo se usan para datos estadísticos o listas, no para diseño.
